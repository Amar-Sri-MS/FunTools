Signing Server

The signing server is a HTTPS server running on port 4443. The URL is "https://[nameofserver]:4443/cgi-bin/signing_server.cgi" It implements the following commands:

GET

GET requests are used to query the server for some information

cmd=modulus

Returns the modulus of a key in various formats.

Arguments:

key (optional argument, default value: 'fpk4')

Specify the label of the key in the HSM. The modulus of this key is returned in the body of the response.

format (optional argument, default value: 'binary')

Specify the format of the modulus in the response.

Possible values are:

  • binary: raw bytes of the modulus

POST

The post request is used to sign an image. There is no "cmd" argument.

Arguments:

img (required)

the image to sign

cert (optional)

A certificate with the public key to use for signing. The certificate will be added to the sign image and the corresponding private key will be used to sign the image.

This correponding private key MUST have been imported into the Server HSM.

customer_cert (optional)

A customer certificate with the public key to use for signing. This is similar to the cert argument but a customer certificate is placed in the different place in the signed image.

key (optional)

Label identifying the key used for signing.

NOTE: All three arguments, cert, customer_cert and key are optional but at least one of them must be provided.

type (required)

the image type. This is a four letter string identifying the type of the image. For example, 'pufr' for PUF-ROM, 'frmw' for Firmware.

version (required)

the image version. This is an number in decimal format.

description (optional)

a description for the image; this is a string limited to 31 characters at most.
